# Data Dictionary

This document describes the main reproducibility artifacts used by the paper and
Software Impacts submission. Paths are relative to the repository root.

## Raw Inputs

### `data/raw/HumanEval.jsonl`

HumanEval benchmark problems.

Key fields:

- `task_id`: HumanEval task identifier, such as `HumanEval/0`.
- `prompt`: function signature and problem prompt.
- `canonical_solution`: reference solution used only for harness validation and LOGIC subtype attribution.
- `test`: benchmark test code.
- `entry_point`: function name under test.

### `data/raw/generations/*.jsonl`

Raw model replies generated by `src/generate.py`.

Key fields:

- `task_id`: HumanEval task identifier.
- `model`: full model identifier, such as `deepseek/deepseek-chat`.
- `sample`: sample index.
- `raw_reply`: original chat-completion reply before code extraction.

## Processed Data

### `data/processed/results.jsonl`

Generated by `src/run_tests.py`.

One row per model/problem/sample execution.

Key fields:

- `task_id`: HumanEval task identifier.
- `model`: full model identifier.
- `sample`: sample index.
- `passed`: boolean pass/fail result.
- `status`: execution status, such as `pass`, `fail_assert`, `exception`, `syntax`, or `timeout`.
- `error_type`: exception or failure type, empty for passing rows.
- `signal`: short failure signal or traceback summary.

Expected row count in the current release: 656.

### `data/processed/labeled.jsonl`

Generated by `src/label.py`.

One row per failing generation.

Key fields:

- `task_id`: HumanEval task identifier.
- `model`: full model identifier.
- `category`: primary bug category.
- `method`: `rule`, `llm`, or `llm_error`.
- `error_type`: failure type inherited from `results.jsonl`.
- `reason`: short rule/LLM explanation.

Expected row count in the current release: 110.

### `data/processed/logic_sublabeled.jsonl`

Generated by `src/sublabel.py`.

One row per primary `LOGIC` failure. The primary subtype annotator sees the problem,
buggy code, and canonical solution.

Key fields:

- `task_id`: HumanEval task identifier.
- `model`: full model identifier.
- `subcategory`: LOGIC subtype label.
- `reason`: short explanation.

Expected row count in the current release: 96.

### `data/processed/logic_sublabeled_blind2.jsonl`

Generated by `src/second_annotator.py`.

One row per primary `LOGIC` failure. The blind second annotator sees the problem and
buggy code, but not the canonical solution, primary label, or primary explanation.

Key fields:

- `task_id`: HumanEval task identifier.
- `model`: full model identifier.
- `subcategory`: blind second subtype label.
- `reason`: short explanation.
- `annotator_model`: model used for the blind annotation.
- `blind`: boolean flag, expected to be `true`.

Expected row count in the current release: 96.

## Result Tables

### `results/tables/pass_rates.csv`

Generated by `src/analyze.py`. Per-model pass rates and sample counts.

### `results/tables/category_frequency.csv`

Generated by `src/analyze.py`. Overall primary bug-category counts and percentages.

### `results/tables/category_by_model.csv`

Generated by `src/analyze.py`. Category-by-model contingency table for RQ2.

### `results/tables/category_by_model_std_residuals.csv`

Generated by `src/analyze.py`. Standardized residuals for the category-by-model
contingency table. Useful for auditing which sparse cells contribute most to the
exploratory chi-square statistic.

### `results/tables/logic_binary_by_model.csv`

Generated by `src/analyze.py`. Two-row companion table that collapses failures into
`LOGIC` and `non_LOGIC` by model.

### `results/tables/logic_binary_chisq.txt`

Generated by `src/analyze.py`. Chi-square statistic, p-value, Cramer's V, and
sparse-cell warning for the LOGIC/non-LOGIC companion test.

### `results/tables/permutation_tests.txt`

Generated by `src/permutation_tests.py`. Deterministic 10,000-iteration
permutation checks for the RQ2 category-by-model and LOGIC/non-LOGIC tables.
These checks are included because the chi-square contingency tables are sparse.

Key fields:

- `seed`: random seed used for deterministic shuffling.
- `n_permutations`: number of random permutations.
- `observed_chi2`: observed chi-square statistic.
- `permutation_p_value`: permutation p-value with add-one finite-simulation smoothing.

### `results/tables/chisq.txt`

Generated by `src/analyze.py`. RQ2 chi-square statistic, p-value, Cramer's V, and
sparse-cell warning.

### `results/tables/logic_subcategory.csv`

Generated by `src/analyze_logic.py`. LOGIC subtype counts and percentages.

### `results/tables/logic_subcat_by_model.csv`

Generated by `src/analyze_logic.py`. LOGIC subtype by model.

### `results/tables/auto_label_agreement.txt`

Generated by `src/kappa.py`. LLM-vs-LLM agreement summary.

### `results/tables/auto_label_confusion.csv`

Generated by `src/kappa.py`. Confusion matrix between primary subtype labels and
blind second subtype labels.

### `results/tables/auto_label_disagreements.csv`

Generated by `src/kappa.py`. Row-level disagreement cases for auditing.

### `results/tables/model_metadata.csv`

Generated by `scripts/generate_model_metadata.py`. Machine-readable model,
provider, generation, and labeler settings for the current artifact.

Key fields:

- `role`: `generation`, `primary_failure_labeler`,
  `primary_logic_subtype_labeler`, or `blind_second_subtype_labeler_default`.
- `model_id`: full hosted model identifier.
- `provider_name`: model gateway/provider name.
- `base_url`: OpenAI-compatible API endpoint used for external model calls.
- `access_date`: recorded date for the current artifact run.
- `benchmark`: data subset for which the row applies.
- `samples_per_problem`: generation sample count, blank for labeler rows.
- `temperature`: model temperature.
- `max_tokens`: maximum completion tokens.

### `results/tables/prompt_inventory.csv`

Generated by `scripts/generate_prompt_inventory.py`. Machine-readable inventory
of prompt templates used for generation, primary failure labeling, primary LOGIC
subtype labeling, and blind second LOGIC subtype labeling.

Key fields:

- `id`: prompt identifier.
- `stage`: pipeline stage where the prompt is used.
- `source`: source file containing the prompt template.
- `template_symbol`: Python symbol name for the prompt template.
- `characters`: prompt-template character count before runtime substitution.
- `sha256`: SHA-256 hash of the prompt template.

### `submission_materials/PROMPT_INVENTORY.md`

Generated by `scripts/generate_prompt_inventory.py`. Human-readable prompt
inventory with full prompt templates and hashes.

## Optional Human Review

### `results/human_eval/review_sheet.csv`

Generated by `scripts/prepare_human_review_sample.py`. Blind 40-item stratified
sample of LOGIC failures for optional human validation. The sheet includes problem
prompts and generated code, but excludes the original LLM subtype labels.

Key fields:

- `id`: review item identifier, such as `HR001`.
- `task_id`: HumanEval task identifier.
- `prompt`: problem prompt.
- `generated_code`: generated code to classify.
- `label_coderA`, `label_coderB`: blank fields for two human reviewers.
- `notes_coderA`, `notes_coderB`: optional reviewer notes.

Expected row count in the current release: 40.

### `results/human_eval/review_key.csv`

Generated by `scripts/prepare_human_review_sample.py`. Key file containing the
original LLM subtype labels and blind second-annotator labels for the sampled
items. It must not be used during blind human review.

### `results/human_eval/CODING_GUIDE.md`

Subtype definitions for human reviewers.

## Reproducibility Notes

- `.env` is intentionally excluded because it may contain API keys.
- Hosted model identifiers may route to provider-current weights; exact regeneration
  depends on provider stability.
- Model and labeler settings are recorded in `config.yaml` and
  `results/tables/model_metadata.csv`.
- Current subtype labels are machine-assigned and should not be described as
  human-validated.
- `scripts/audit_release.py` checks current row counts, key statistics, manuscript
  consistency, and obvious secret-looking key patterns.
